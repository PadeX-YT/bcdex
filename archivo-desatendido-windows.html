🛠️ Guía completa para optimizar Windows 11 durante la instalación

Instalar Windows 11 no tiene por qué ser una tarea lenta ni repetitiva. Con las herramientas adecuadas, puedes hacer que la instalación se realice prácticamente sola y además dejar el sistema limpio, optimizado y sin rastros de bloatware desde el primer arranque.

En esta guía te explico cómo funciona esa automatización, cómo puedes usar un archivo desatendido para personalizar el instalador de Windows, y cómo BCDeX simplifica todo ese proceso incluso si no tienes experiencia con XML.

¿Qué es un archivo desatendido?

El archivo desatendido, llamado autounattend.xml, es un documento en formato XML que le indica al instalador de Windows todo lo que debe hacer automáticamente, sin intervención del usuario.

Este archivo puede personalizar:

  • Idioma, teclado y formato regional
  • Aceptación de términos y condiciones
  • Edición de Windows (Home, Pro, Education...)
  • Configuración de particiones (avanzado)
  • Tipo de cuenta (local o Microsoft)
  • Nombre del equipo y usuario
  • Saltar Cortana, ubicación, y otras opciones de privacidad
  • Eliminar apps sugeridas (bloatware)
  • Ejecutar scripts o tareas tras finalizar la instalación
  • Y mucho más…

La instalación se convierte en una experiencia completamente automática y personalizada.

¿Cómo se usa el archivo desatendido?

Solo tienes que guardarlo con el nombre exacto autounattend.xml y colocarlo en la raíz del medio de instalación (ya sea un USB o una partición interna).

Cuando arranca el instalador de Windows, detecta ese archivo automáticamente y lo sigue paso a paso. No es necesario tocar nada. Literalmente, puedes dejar que Windows se instale solo mientras tú te tomas un café.

¿Y qué hace BCDeX con esto?

BCDeX automatiza toda esa parte por ti. Cuando eliges una ISO de Windows 10 u 11, la app no solo extrae los archivos y configura el arranque desde disco, sino que además:

🔧 Genera automáticamente un archivo desatendido personalizado, pensado para:

  • Eliminar bloatware
  • Omitir preguntas innecesarias
  • Saltarse la cuenta Microsoft
  • Activar cuenta local
  • Desactivar sugerencias y servicios molestos
  • Acelerar la instalación inicial

Este XML ya está optimizado para la mayoría de usuarios. Solo con usar la app, ya estás obteniendo una instalación limpia, rápida y afinada sin complicarte la vida.

Además, BCDeX evita que necesites un USB. Todo se instala desde el propio disco duro, utilizando una partición dedicada. Más cómodo, más rápido y más moderno.

¿Y si quiero personalizar aún más?

Si quieres ir más allá, puedes crear tu propio archivo desatendido y reemplazar el que genera BCDeX.

Por ejemplo, podrías:

  • Usar un nombre de equipo o de usuario distinto
  • Establecer una contraseña
  • Elegir la edición exacta de Windows
  • Crear múltiples cuentas
  • Ejecutar un script de post-instalación
  • Activar características opcionales como Hyper-V, WSL o SMB1
  • Preconfigurar redes o tareas programadas

Es un mundo muy potente para quienes quieran tener su instalación 100% adaptada a sus necesidades.

¿Cómo crear tu propio archivo desatendido?

Si decides crear tu propio archivo autounattend.xml, no necesitas ser un experto. Aquí tienes los pasos básicos:

  1. Abre un editor de texto como Bloc de notas, Notepad++ o Visual Studio Code.
  2. Escribe la estructura básica XML con los bloques necesarios para cada fase: configuración regional, OOBE, creación de usuarios, etc.
  3. Guárdalo con el nombre exacto: autounattend.xml (sin .txt).
  4. Colócalo en la raíz del medio de instalación (la carpeta donde están los archivos extraídos de la ISO).
  5. Al iniciar la instalación de Windows, el sistema detectará automáticamente ese archivo y aplicará la configuración.

Y si prefieres algo más sencillo, recuerda que BCDeX ya genera un archivo optimizado por ti. Pero si necesitas personalizar más, ahora sabes cómo hacerlo.

Ejemplo real: cómo se ve por dentro un autounattend.xml 

<unattend xmlns="urn:schemas-microsoft-com:unattend">
  <settings pass="oobeSystem">
    <!-- Configuración durante la primera ejecución del sistema (OOBE) -->
    <component name="Microsoft-Windows-Shell-Setup" processorArchitecture="amd64"
               publicKeyToken="31bf3856ad364e35" language="neutral" versionScope="nonSxS"
               xmlns:wcm="http://schemas.microsoft.com/WMIConfig/2002/State"
               xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
      <OOBE>
        <!-- Oculta la pantalla de aceptación de licencia -->
        <HideEULAPage>true</HideEULAPage>

        <!-- Establece el tipo de red como "Trabajo" -->
        <NetworkLocation>Work</NetworkLocation>

        <!-- Omite la configuración inicial del sistema -->
        <SkipMachineOOBE>true</SkipMachineOOBE>

        <!-- Omite la creación guiada de usuarios -->
        <SkipUserOOBE>true</SkipUserOOBE>
      </OOBE>

      <UserAccounts>
        <LocalAccounts>
          <LocalAccount>
            <Password>
              <Value>1234</Value>
              <PlainText>true</PlainText>
            </Password>

            <!-- Nombre visible para la cuenta -->
            <DisplayName>Admin</DisplayName>

            <!-- Nombre del usuario local -->
            <Name>admin</Name>

            <!-- Grupo al que se añade (Administradores) -->
            <Group>Administrators</Group>
          </LocalAccount>
        </LocalAccounts>
      </UserAccounts>
    </component>
  </settings>
</unattend>

Este ejemplo básico crea una cuenta local llamada "admin", salta toda la configuración inicial y evita que el usuario tenga que hacer clic en nada. Ideal para instalaciones rápidas y limpias.
